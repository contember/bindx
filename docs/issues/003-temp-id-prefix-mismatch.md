# 003: Temp ID prefix mismatch in JSX proxy

**Severity:** Critical
**Category:** Bug
**Reported by:** gamma

## Location

- `packages/bindx-react/src/jsx/proxy.ts:523` (generates `new_*` prefix)
- `packages/bindx/src/store/SnapshotStore.ts:72-73` (checks `__temp_*` prefix)
- `packages/bindx/src/persistence/BatchPersister.ts:480` (checks `__temp_*` prefix)

## Description

The JSX proxy `add()` function generates temporary IDs with a `new_` prefix, but the rest of the codebase expects temporary IDs to start with `__temp_`.

```typescript
// proxy.ts — generates wrong prefix
const newId = `new_${Date.now()}_${Math.random().toString(36).substring(2, 11)}`

// SnapshotStore.ts — expects __temp_ prefix
export function isTempId(id: string): boolean {
    return id.startsWith('__temp_')
}
```

## Impact

- Entities created via `<HasMany>.add()` in JSX components are not recognized as temporary
- Persistence code may attempt to update instead of create these entities
- `isTempId()` returns `false` for IDs generated by the JSX proxy
- Temp ID filtering in `collectCreateData` won't exclude the ID field

## Fix

Use `store.createEntity()` in the proxy's `add()` function to generate proper `__temp_` IDs, consistent with the rest of the system.
